3. Singleton Pattern penting karena:
- Membuat database thread-safe (dibantu @Volatile dan synchronized) 
- Mencegah race condition antar-thread.
- Menghindari overhead pembuatan banyak instance database (sangat mahal secara proses).
- Menjaga konsistensi data selama aplikasi berjalan.
Tanpa Singleton, database bisa dibuat banyak kali dan menyebabkan crash, inkonsistensi, dan konsumsi resource yang besar.

4. Repository Pattern penting karena Repository Pattern berfungsi sebagai abstraction layer antara API (network) dan Room database (local storage). Repository Pattern membuat semuanya menjadi lebih sistematis dalam hal management, sehingga Repository adalah pola yang disarakan atau bahkan wajib untuk aplikasi yang menggunakan API dan Room.

5. Selain WorkManager, beberapa cara lain untuk me-refresh database adalah:
- Manual refresh di UI (kekurangan: tidak otomatis)
- Timer/Handler di Activity (kekurangan: tidak berjalan jika aplikasi ditutup dan tidak efisien)
- Foreground Service (kekurangan: otomatis tapi sangat berat (resource intensive))
- BroadcastReceiver (CONNECTIVITY_CHANGE, BOOT_COMPLETED) (kekurangan: tidak menjadwalkan pekerjaan secara periodik)
- AlarmManager (kekurangan: kurang akurat dan tidak sefleksibel WorkManager)
